# 4.4 URDF: Defining the Humanoid Robot's Kinematics

## 4.1 Introduction to Robot Description Formats
Robot software ko robot ke physical structure aur geometry ko jan'na zaroori hai. Iske liye Robot Description Formats use hote hain.

**URDF (Unified Robot Description Format):** XML-based format, primarily used in ROS, for describing a robot's kinematic and dynamic structure (mass, inertia). URDF aik single rigid tree structure describe karta hai (yaani no closed loops allowed).

**SDF (Simulation Description Format):** Gazebo aur Igniton/DART jaise advanced physics simulators ka native format. SDF URDF se zyada powerful hai; yeh closed-loop mechanisms, environment objects, lights, aur terrains ko bhi describe kar sakta hai.

## 4.2 Kinematics and Dynamics: Links and Joints
URDF robot ki body ko do main elements se define karta hai:

`link`: Represents a rigid body part of the robot (e.g., the torso, thigh, forearm). Har link ki mass, inertia, aur visual geometry hoti hai.

`<joint>`: Connects two links (parent aur child). Joint hi decide karta hai ki do links aapas mein kaise move kar sakte hain.

**Types of Joints:**

* **Revolute:** Rotation around one axis (Motorized joints).
* **Prismatic:** Linear slide along one axis.
* **Fixed:** No movement, used to attach a sensor rigidly to a link.

## 4.3 Visual, Collision, and Inertial Elements
Har `<link>` ke andar teen important sub-elements hote hain:

* `<visual>`: Specify karta hai ki robot simulator (e.g., Rviz, Gazebo) mein kaisa dikhega (color, texture, mesh file path).
* `<collision>`: Specify karta hai ki robot ke kaunse hisse physics simulation mein contact detect karenge. (Aksar collision geometry visual geometry se simple rakhi jaati hai).
* `<inertial>`: Specify karta hai ki robot physics simulation mein kaise behave karega (mass aur inertia tensor). Yeh element dynamics ke liye bahut zaroori hai.

## 4.4 Practical URDF Example: Two-Link Arm
Aapka humanoid robot bahut complicated URDF use karega, lekin basic concept yeh hai:

```xml
<?xml version="1.0"?>
<robot name="two_link_arm">

    <link name="base_link">
        <inertial>...</inertial>
        <visual>
            <geometry><box size="0.1 0.1 0.1"/></geometry>
        </visual>
    </link>

    <joint name="shoulder_joint" type="revolute">
        <parent link="base_link"/>
        <child link="link_1"/>
        <axis xyz="0 0 1"/>
        <limit effort="100" velocity="0.5" lower="-3.14" upper="3.14"/>
        <origin xyz="0 0 0.1" rpy="0 0 0"/>
    </joint>

    <link name="link_1">
        <inertial>...</inertial>
        <visual>
            <geometry><cylinder radius="0.05" length="0.5"/></geometry>
        </visual>
    </link>

</robot>
The Kinematic Chain is the sequence of links and joints that defines the robot's structure. For a humanoid arm, the chain is: base_link -> shoulder_joint -> link_1 -> elbow_joint -> link_2 -> wrist_joint -> end_effector. Forward Kinematics is the process of calculating the end-effector's position given all joint angles.

Key Concept Summary
URDF is the primary format for describing a robot's structure using links (rigid bodies) and joints (connections). URDF describes the kinematic chain. SDF is used for advanced physics simulation and environmental elements. Every link requires <visual>, <collision>, and <inertial> elements to define its appearance, interaction boundaries, and dynamic behavior (mass and inertia). This definition is fundamental for both simulation and real-world control software.
